MEDIUM PRIORITY FINDINGS
================================================================================

FINDING 1: Fallback Payment Hash Generation
Location: utils/NostrConnectUtils.ts:546-566, 589-592
Severity: MEDIUM | Category: Logic Correctness

Issue: When primary payment hash sources unavailable, falls back to:
  sha256(`zeus-nwc-fallback:${timestamp}-${amount}`)
  
Risk: Two payments at same timestamp with identical amount produce same hash
      → Activity dedup sees as single transaction (rare edge case)
      → User sees one transaction instead of two

Recommendation: Include millisecond precision or payment.id in fallback key
Acceptable As-Is: Yes (99% case covered, fallback is defensive not primary)

---

FINDING 2: Budget Renewal - Time Zone Boundary
Location: models/NWCConnection.ts:290-310
Severity: MEDIUM | Category: Logic Correctness

Issue: "daily" budget renewal uses absolute 24-hour intervals (86,400,000 ms)
       User might expect calendar-aligned reset (e.g., 23:00 their local time)
       Instead resets based on creation timestamp + 24h UTC

Risk: Low—behavioral, not data corruption. May surprise users.
      This is CORRECT design (simpler, no DST edge cases)

Recommendation: Document in UI. Current design is safer than calendar-aligned.
Acceptable As-Is: Yes (simple, predictable, no date computation edge cases)
